## Highlights

- **튜토리얼 시스템 안정화**: stuck 검출 + recovery, peek-close 신규 step, cold-start step 복원, 모달 애니메이션 (fade-slide-fade), 핵심 키워드 강조 등 14-step 흐름 전반의 흔들림 해소.
- **Dynamic Type 앱 정책**: 시스템 텍스트 크기 설정과 무관하게 앱 자체 4단계 (small / medium / large / extraLarge) 로 UI 일관성 확보.
- **카메라 재진입 버그 수정**: 설정 등 push 후 돌아왔을 때 BeforeCamera 프리뷰가 멈추던 lifecycle 누락을 정공으로 보강.

## Added

- 튜토리얼 step 8 신설 — `afterCameraStripPeekClose` (미리보기 영역 밖 탭 실습) + AfterCameraStrip preview 의 onAppear/onDisappear 기반 자동 advance (`tutorial`)

- 튜토리얼 메시지 모달 — max-height 클램프 + ViewThatFits 폴백 + step 9 (afterCameraGuide) 위로 220pt anchorGap (`tutorial`)

## Fixed

- 설정·내보내기 등 push 후 pop 으로 BeforeCamera 로 돌아왔을 때 프리뷰가 멈추던 버그 — `didStartCameraSession` 가드가 reset 되지 않아 `session.start()` 가 누락되던 lifecycle. `onAppear` 마다 idempotent 재진입 호출 추가 (`camera`)
- 튜토리얼 cold-start step 복원 — `tutorialCurrentStepRawValue` 영속화 + `TutorialCoordinator.resume(at:)` + `normalizeForResume` (peekClose / afterCameraInProgress → afterCameraGuide) (`tutorial`)
- 튜토리얼 anchor 없을 때 centered modal fallback (`tutorial`)
- `afterCamera resume` 시점 카메라 session 충돌 — `BeforeCameraView.shouldStartCameraSession` 가드 도입 (`camera`)
- ExportSettingsView body type-check 시간 단축 + ScrollViewReader 로 step 진입 시 카드 자동 scroll (`export`)
- HomeBottomBarHost / AfterCameraView 튜토리얼 step 변화에 대한 selection·target 자동 정비 (`tutorial`)
