## Highlights

- **구독 결제 직후 paywall 자동 닫힘 + 광고 즉시 제거**: 실제 결제 환경에서 paywall 이 한참 안 닫히고 광고도 그대로 남아 앱을 재실행해야 PRO 상태가 반영되던 문제 해결. Apple `SubscriptionStoreView` 의 `.onInAppPurchaseCompletion` 직접 콜백 경로를 추가해 결제 즉시 유료 상태를 재조회하고, 그 시점부터 paywall · 배너 · 네이티브 광고 모두 즉시 반응.
- **Paywall 이 앱 텍스트 크기 설정을 따름**: 디바이스 시스템 텍스트 크기를 매우 크게 쓰는 사용자가 paywall 진입 시 레이아웃이 깨지던 문제 해결. iOS 17 trait override 로 윈도우 씬 단위 텍스트 크기를 앱 설정에 동기화해 Apple `SubscriptionStoreView` 까지 우리 앱 4단계 텍스트 크기 정책을 따름. ATT · 권한 · Face ID 결제 시트 등 시스템 외부 UI 는 OS 제약상 변동 없음.
- **설정의 텍스트 크기·테마 변경이 앱 재실행 없이 즉시 반영**: 그동안 picker 에서 값을 바꿔도 앱을 종료-재실행해야 적용되던 문제 해결. 텍스트 크기 picker 는 선택 후 자동으로 닫히지 않게 바꿔 여러 크기를 직접 비교한 뒤 화면을 빠져나갈 수 있도록 조정.

## Changed

## Fixed

- 실제 결제 환경의 paywall 에서 결제 완료 후에도 화면이 자동으로 닫히지 않고 광고가 그대로 남던 문제 수정. Apple `SubscriptionStoreView` 의 `.onInAppPurchaseCompletion` 콜백 경로를 추가해 결제 직후 즉시 `transaction.finish` + `subscriptionStore.refresh` + membership reconcile 을 수행. `isPro` 가 true 로 바뀌는 시점에 기존 dismiss / 광고 숨김 wiring 이 자동 발동. 기존 외부 이벤트 리스너 (갱신·환불·다른 기기 변경) 는 그대로 유지. (`paywall`)
- Paywall 텍스트 크기가 디바이스 시스템 설정을 따라가 앱 4단계 설정을 무시하고 레이아웃이 깨지던 문제 수정. iOS 17 `UIWindowScene.traitOverrides.preferredContentSizeCategory` 를 앱 텍스트 크기에 동기화해 SwiftUI + in-process UIKit + Apple `SubscriptionStoreView` 까지 동일하게 우리 앱 정책을 따름. 앱 부팅 시 1회 + 설정 변경 시 재적용. ATT · 권한 prompt · 결제 Face ID 시트 등 out-of-process 시스템 UI 는 iOS 보안 정책상 영향 범위 밖. (`paywall`)
- 설정 → 텍스트 크기 / 테마 변경이 앱 재실행해야 반영되던 문제 수정. `AppSettings` 의 `theme`, `appTextSize` 를 `UserDefaults` wrapping computed property 에서 stored property + `didSet` 패턴으로 전환해 `@Observable` 추적을 정상화. 변경 즉시 `PairShotApp` 의 `.preferredColorScheme` · `.dynamicTypeSize` 가 재평가되어 앱 전체에 즉시 반영. (`settings`)
